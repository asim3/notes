# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A static single-page reference guide (asim3.github.io/notes/). There is no build step, no package manager, and no framework. All content lives in `data/` as Markdown files.

## Running locally

```bash
docker compose up   # serves on http://localhost:3000
```

Or open `index.html` directly in a browser (fetch calls require a server; raw `file://` won't work).

## How the site works

- `index.html` bootstraps the app; `static/js/main.js` handles all navigation via `fetch()`.
- On load, `fetch_path()` reads `?path=` from the URL and fetches the corresponding file under `data/`.
- Each directory gets an `init.html` navigation file (list of buttons) generated by `scripts/update.sh`.
- `init.html` files are regenerated automatically via GitHub Actions (`.github/workflows/update-init-files.yaml`) on every push to `main`, which runs `make`.

## Adding or updating content

1. Add/edit `.md` files inside `data/<topic>/`.
2. New subdirectories need an `index.md` to serve as the directory landing page.
3. Run `./scripts/update.sh` locally (or rebuild the Docker image) to regenerate all `init.html` nav files and `data/search-index.json` — otherwise new files won't appear in the sidebar navigation or search results.

## Search

Every page includes a search bar powered by `data/search-index.json` — a JSON array of all `.md` file paths under `data/`. It is regenerated automatically by `scripts/update.sh` (and therefore by CI on every push to `main`). No manual step is needed beyond running `update.sh` after adding new files.

## Custom content tags

Files can use either standard Markdown or a custom tag syntax parsed by `parse_tags()` in `main.js`:

```
..title.. Page Title
..sub_title.. Section heading
..text.. Paragraph text
..code.. inline code block
..link path/to/file.md Button label
..raw.. raw HTML passthrough
```

Arabic variants exist: `..عنوان..`, `..فرعي..`, `..نص..`, `..رابط..`, `..برمجة..` (triggers RTL layout).

## CI / deployment

The `makefile` is used by CI only — it checks out the `production` branch, runs `update.sh`, and force-pushes. Do not run `make` locally for content editing; just edit the markdown files directly.
